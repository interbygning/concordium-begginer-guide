1. Server Preparation

1.1. Preparing the environment

Firstly we need to prepare the environment for work, to do this we will do everything in a separate screen and write down the variables that will be used later.
screen -S concordium
echo 'export PATH=$PATH:/home/concordium/concordium-software' >> $HOME/.bashrc
echo 'export IP=$(curl ifconfig.me)' >> $HOME/.bashrc
source ~/.bashrc

1.2. Installing the required packages

Next, we need to install the necessary packages, for this we run the following commands:
apt update
apt install apt-transport-https ca-certificates curl gnupg-agent software-properties-common wget screen -y
Now we need to install Docker, as it is required to run the Concordium node. To do this, run the following commands:
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
sudo apt-key fingerprint 0EBFCD88
add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"
apt update
apt install docker-ce docker-ce-cli containerd.io -y
Let's check if everything was installed correctly:
systemctl status docker

If so, you will see the same as in the screenshot. If not, repeat the steps above.

1.3. Create a User

Now we need to create a special user as the Concordium node will not work under the root. To do this, follow these steps:
useradd -m concordium --shell /bin/bash
echo "concordium:onehundredsecurepwd*100" | chpasswd
You can choose whichever username and password you like, suggested here is just an example. The concordium username is used for convenience, if you need something else - you can set a different value, but then take that into account when performing the next steps.
Now let's assign the necessary groups to the user:
usermod -aG sudo concordium
groupadd docker
usermod -aG docker concordium
Now we need to prepare the user's environment and log in under the created account, so we run the following commands:
cd /home
chown concordium: concordium
su --login concordium

1.4. User setting

After changing the user, we need to re-add the variables, for this we execute:
echo 'export PATH=$PATH:/home/concordium/concordium-software' >> $HOME/.bashrc
echo 'export IP=$(curl ifconfig.me)' >> $HOME/.bashrc
source ~/.bashrc

2. Installing a node

2.1. Downloading nodes

Now we need to download the software of the Concordium node, for this we execute:
wget -O concordium.tar.gz https://client-distribution-testnet.concordium.com/concordium-software-linux_0.4.8-0.tar.gz
tar -xf concordium.tar.gz
cd concordium-software

2.2. Running a node

Launch the node:
./concordium-node
If everything is okay, then you will see the following:

Here you need to enter the name of your node, enter any name - you need this to find your node in the explorer.
After that, you should have a window as shown above, and then your Concordium node will work in it.

3. Working with Concordium ID application

3.1. Installing the application

3.1.1. Installation on Android
To install, you need a phone with Android version 8 or higher.
  • Download Concordium ID APK
  • You may see warnings about downloading the .apk file, if this happens, click OK.
  • After launching the .apk file, you may see a message stating that the installation of applications from unknown sources is prohibited in your settings.
    In order to allow the installation, go to the settings and confirm the installation of the application, and then go back.
  • Now you will be asked if you want to install the application, click Install.
3.1.2. Installation on iOS
To install on iOS, we need iPhone with iOS 13+ and TestFlight app.
  • Install TestFlight via AppStore
  • Join the testing programm (this step is recommended to be done directly from the phone)
  • Open TestFlight app and set Concordium ID.

3.2. Запуск приложения

Launch the application, click Yes, lets go! and set the password:
Then click 'I want to create my initial account', then Okay, got it!

3.3. Account creation

We need to indicate:
  • Account name (hereinafter ConcordiumBegginerGuide will be used as the account name)
  • Your name
  • KYC provider
At the moment, you can go through KYC both using real documents and without them. To pass KYC without documents, select Notabene Development.

After you go through KYC, you will see your account and balance on it.

Now you need to wait a while (while the KYC confirmation is in progress), after which you will be able to request a balance.

3.4. Request a balance

Click on your account name. You will see a window in which you can request 100 GTU (you will actually receive 2000 GTU, the value of 100 is left from the last testnet), ask and wait for receipt.
After the tokens arrive on your balance, you can proceed to the next step.

3.5. Export account

The account export procedure is necessary so that we can fully use the node. In order to export your account, follow these steps:
  • In the app, click More in the bottom right corner
  • Click Export
  • Click Continue
  • Enter the password that you will use later when working with an account (in the node)
  • Scroll down, select Save to Files, save
Now you need to transfer the export.concordiumwallet file to your computer and then to the server. To upload a file to the server, use an FTP client (for example FileZilla or WinSCP).
After connecting to the server via FTP, save your keys in a convenient place for you (in the future we will use the path /home/concordium/jsons/export.concordiumwallet as the path to your key file, it is recommended to save them in the same place).

4. Adding an account

This step assumes that you have already exported an account and it is located in the path / home / concordium / jsons / named export.concordiumwallet (i.e. the full path to the file looks like / home / concordium / jsons / export .concordiumwallet)

4.1. Import account

Import the account with the command:
concordium-client config account import /home/concordium/jsons/export.concordiumwallet --name ConcordiumBegginerGuide --grpc-ip $IP

4.2. Creating a baker

Now we need to make the keys for the baker, for this we run the commands:
mkdir /home/concordium/keys
/home/concordium/keys
concordium-client baker generate-keys /home/concordium/keys/key.json
Now we need to wait for the node to sync, it will take some time.
To check the status of a node, you need to find it in the explorer (in our case, it is called ConcordiumBegginerGuide) and compare the 'Height of best block' value with another, already synchronized node (for example, a node with an uptime of more than a day).

4.3. Adding a baker

When the node is synchronized, proceed to the next step - add a baker:
concordium-client baker add /home/concordium/keys/key.json --sender ConcordiumBegginerGuide --stake 1000 --out ~/.local/share/concordium/baker-credentials.json --no-restake --grpc-ip $IP
As a result, you should see the picture like this:

For the Baker ID to appear in the explorer, you need to wait 2 epochs (~2 hours) and restart the node, after which the Baker ID will appear.
Let's check if we are a baker. To do this, enter the command:
concordium-client raw GetNodeInfo --grpc-ip $IP
'Baker committee member' must be True.
If 'False' - restart the node or wait. If after this 'Baker committee member' is still 'False' - check for the presence of baker-credentials.json at /home/concordium/.local/share/concordium, for this you'll need to run the command:
cat /home/concordium/.local/share/concordium/baker-credentials.json
The output should look like this:

If you get an error, then repeat the steps above (p. 4.3)
Congratulations, you've launched the Concordium node!
